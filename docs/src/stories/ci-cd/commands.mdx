<Title>⚙️ Commands</Title>

The CI/CD workspace provides a comprehensive set of automation commands organized into logical categories. Each command is designed to target specific workspaces and perform coordinated operations across the monorepo.

## Command Categories

| Command           | Purpose            | Scope                  |
|-------------------|-------------------|------------------------|
| `build`           | Production builds | App only               |
| `lint:code:run`   | Code linting      | Most workspaces        |
| `lint:staged:fix` | Pre-commit fixes  | All workspaces         |
| `lint:styles:run` | Style linting     | UI workspaces          |
| `lint:types:run`  | Type checking     | TypeScript workspaces  |
| `test:ui:run`     | UI testing        | Component workspaces   |
| `test:unit:run`   | Unit testing      | Logic workspaces       |

## Usage Tips

- **Development Workflow**: Use `lint:staged:fix` as a pre-commit hook
- **Quality Gates**: Run `lint:types:run` before merging
- **Testing Strategy**: Combine `test:unit:run` and `test:ui:run` for full coverage
- **Production Readiness**: Execute `build` after all quality checks pass
