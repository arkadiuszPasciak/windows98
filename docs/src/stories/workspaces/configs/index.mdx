## Why?

The philosophy behind the `@windows98/configs` package is centered around providing centralized, reusable configuration files for all development tools and build processes across the Windows98 monorepo. This design ensures consistency, maintainability, and easy updates to linting rules, TypeScript settings, testing configurations, and other development tools without duplicating configuration files across workspaces.

## What's Included?

The configs workspace provides shared configuration for:

- **🔍 Biome** - Code formatting and linting
- **🎭 Lint Staged** - Pre-commit code quality checks
- **🎪 Playwright** - End-to-end and component testing
- **📦 PNPM** - Package management
- **🎨 Stylelint** - CSS/SCSS linting and formatting
- **📘 TypeScript** - Type checking and compilation
- **⚡ Vitest** - Unit and integration testing
