  --ds-background-color-secondary: var(--ds-color-white100);
  --ds-background-color-tertiary: var(--ds-color-grey200);
  --ds-background-color-modal-header: linear-gradient(
    90deg,
    var(--ds-color-blue200),
    var(--ds-color-blue300)
  );
  --ds-background-color-pulpit: var(--ds-color-blue400);

  --ds-text-color-primary: var(--ds-color-black300);
  --ds-text-color-secondary: var(--ds-color-white100);
  --ds-text-color-tertiary: var(--ds-color-black300);

  --ds-border-color-secondary: var(--ds-color-grey400);
  --ds-border-color-tertiary: var(--ds-color-white100);

## Color Usage Guidelines

### Do's ‚úÖ

1. **Use semantic variables** for theme-aware colors
   ```css
   color: var(--ds-text-color-primary);
   background: var(--ds-background-color-primary);
   ```

2. **Test both themes** to ensure proper contrast and visibility

3. **Use base color variables** for static elements that don't change with themes

### Don'ts ‚ùå

1. **Don't use hardcoded colors** that break theme switching
   ```css
   /* ‚ùå Bad */
   color: #222222;
   
   /* ‚úÖ Good */
   color: var(--ds-text-color-primary);
   ```

2. **Don't override system colors** without maintaining Windows 98 aesthetics

3. **Don't ignore accessibility** - ensure sufficient contrast ratios

## Custom Color Extensions

### Adding Custom Colors

```css
:root {
  /* Custom color additions */
  --ds-color-custom-green: #00ff00;
  --ds-color-custom-yellow: #ffff00;
}

/* Theme-specific usage */
:root[theme="light"] {
  --ds-text-color-success: var(--ds-color-custom-green);
  --ds-text-color-warning: var(--ds-color-custom-yellow);
}
```

### Brand Color Integration

```css
/* Integrate brand colors while maintaining Windows 98 aesthetic */
:root {
  --ds-color-brand-primary: #ff6b35;
  --ds-color-brand-secondary: #004e89;
}

/* Use in specific components */
.brand-button {
  background: var(--ds-color-brand-primary);
  color: var(--ds-color-white100);
  box-shadow: var(--ds-border-raised);
}
```
