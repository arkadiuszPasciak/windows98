Semantic color tokens that adapt based on the selected theme (light/dark) for the Windows98 design system.

## Overview

Semantic colors provide meaning-based color assignments that automatically adapt between light and dark themes. These tokens abstract the underlying color values and provide consistent naming conventions for different UI purposes.

## 3D Border Effects

Semantic 3D border tokens that create the authentic Windows 98 raised and sunken effects.

### 3D Border Components

```css
/* 3D Border Inner Effects */
--ds-border-field-inner: /* Inner layer of form field borders */
--ds-border-raised-inner: /* Inner layer of raised elements */
--ds-border-sunken-inner: /* Inner layer of sunken elements */

/* 3D Border Outer Effects */
--ds-border-field-outer: /* Outer layer of form field borders */
--ds-border-raised-outer: /* Outer layer of raised elements */
--ds-border-sunken-outer: /* Outer layer of sunken elements */

/* Combined 3D Borders */
--ds-border-field: /* Complete form field border effect */
--ds-border-raised: /* Complete raised element effect */
--ds-border-sunken: /* Complete sunken element effect */
```

### Light Theme 3D Borders

```css
:root[theme="light"] {
  /* Field borders (sunken appearance) */
  --ds-border-field-inner: inset -1px -1px var(--ds-color-white100), 
                           inset 1px 1px var(--ds-color-grey200);
  --ds-border-field-outer: inset -2px -2px var(--ds-color-grey900), 
                           inset 2px 2px var(--ds-color-black200);

  --ds-border-raised-outer: inset -2px -2px var(--ds-color-grey200), 
                            inset 2px 2px var(--ds-color-grey900);
```

## Usage Examples

### Component Implementation

```css
/* Button using semantic colors */
.ds-button {
  background-color: var(--ds-background-color-secondary);
  color: var(--ds-text-color-primary);
  box-shadow: var(--ds-border-raised);
}

.ds-button:hover {
  background-color: var(--ds-background-color-secondary);
}

.ds-button:active {
  box-shadow: var(--ds-border-sunken);
}

/* Input field using semantic colors */
.ds-input {
  background-color: var(--ds-background-color-secondary);
  color: var(--ds-text-color-primary);
  box-shadow: var(--ds-border-field);
  border: 1px solid var(--ds-border-color-secondary);
}

/* Modal header with gradient */
.ds-modal-header {
  background: var(--ds-background-color-modal-header);
  color: var(--ds-text-color-secondary);
}

/* Panel with semantic background */
.ds-panel {
  background-color: var(--ds-background-color-primary);
  border: 1px solid var(--ds-border-color-primary);
  box-shadow: var(--ds-border-raised);
}
```

## Semantic Color Guidelines

### Do's ‚úÖ

1. **Use semantic tokens** instead of direct color references
   ```css
   /* ‚úÖ Semantic approach */
   background-color: var(--ds-background-color-primary);
   
   /* ‚ùå Direct color reference */
   background-color: var(--ds-color-grey800);
   ```

2. **Follow the hierarchy** for text colors
   ```css
   /* ‚úÖ Proper hierarchy */
   .title { color: var(--ds-text-color-primary); }
   .subtitle { color: var(--ds-text-color-secondary); }
   .caption { color: var(--ds-text-color-tertiary); }
   ```

3. **Use appropriate 3D borders** for element states
   ```css
   /* ‚úÖ Semantic border usage */
   .button { box-shadow: var(--ds-border-raised); }
   .input { box-shadow: var(--ds-border-field); }
   .pressed { box-shadow: var(--ds-border-sunken); }
   ```

### Don'ts ‚ùå

1. **Don't hardcode colors** that should be semantic
   ```css
   /* ‚ùå Hardcoded theme-specific color */
   background-color: #c0c0c0;
   
   /* ‚úÖ Semantic color that adapts */
   background-color: var(--ds-background-color-secondary);
   ```

2. **Don't mix semantic levels** inappropriately
   ```css
   /* ‚ùå Wrong semantic level */
   .main-content { color: var(--ds-text-color-tertiary); }
   
   /* ‚úÖ Appropriate semantic level */
   .main-content { color: var(--ds-text-color-primary); }
   ```

3. **Don't bypass the 3D border system**
   ```css
   /* ‚ùå Manual border construction */
   box-shadow: inset 1px 1px white, inset -1px -1px gray;
   
   /* ‚úÖ Semantic 3D border */
   box-shadow: var(--ds-border-raised);
   ```
