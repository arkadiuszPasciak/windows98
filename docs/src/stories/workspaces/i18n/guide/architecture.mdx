<Title>🏗️ Architecture</Title>

Understanding the structure and design of the `@windows98/i18n` package for effective internationalization management.

## Package Structure

```
core/internationalization/i18n/
├── package.json              # Package configuration
├── tsconfig.json             # TypeScript configuration
├── README.md                 # Package documentation
├── @linter/                  # Linting configurations
│   ├── biome.config.json
│   └── lint-staged.config.cjs
├── configs/                  # Build and development configs
└── src/                      # Source code
    ├── locales/              # Translation files
    │   ├── index.config.ts   # Main export configuration
    │   ├── en/               # English translations
    │   │   └── index.ts
    │   ├── de/               # German translations
    │   │   └── index.ts
    │   ├── pl/               # Polish translations
    │   │   └── index.ts
    │   └── micro-frontends/  # Micro-frontend specific translations
    │       ├── en/
    │       │   ├── index.ts
    │       │   ├── mf-calculator.ts
    │       │   ├── mf-notepad.ts
    │       │   ├── mf-calendar.ts
    │       │   └── ...
    │       ├── de/
    │       │   ├── index.ts
    │       │   ├── mf-calculator.ts
    │       │   └── ...
    │       └── pl/
    │           ├── index.ts
    │           ├── mf-calculator.ts
    │           └── ...
    └── types/                # TypeScript type definitions
        └── index.ts
```
